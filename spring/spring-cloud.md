# Spring Cloud

是一个大杂烩，集成了各种开源组件，拼凑出一个微服务框架。主要包含如下组件:

* 服务治理（注册与发现）,可选Consul和Eureka. 主要功能就是注册与发现，维护一份服务的清单，包括IP，状态。通过心跳来感知状态。
* 负载均衡。Ribbon。常见的策略是循环。
* 断路器 Hystrix. 当某个服务响应过慢，导致熔断，后续的请求会快速返回(fallback).
* 申明式调用(Feign Client). 通过接口和注解，可以直接进行RPC。 
* 分布式配置 ConfigServer。从git读取配置。
* 业务网关 ZUUL. 业务网关主要功能是分流，权限控制等。跟Nginx相比，它能集成Ribbon，Hystrix，从而做到负载均衡和容错。Nginx作为反向代理，它必须使用固定的IP地址，而微服务的IP是动态的。

另外需要说明的是，负载均衡和容错（断路器）都是在Client里面实现的。
